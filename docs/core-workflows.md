# Clearance — Core Workflows

This document defines the primary linear workflows.
No branching, exceptions, or UI considerations included.

---

## 1. Permit Lifecycle

1. User creates Permit
2. Permit assigned to Jurisdiction
3. Required Documents attached
4. Inspections scheduled (if required)
5. Inspections completed
6. Approval requested
7. Approval granted or denied
8. If approved → Invoice generated
9. Permit marked as Active or Closed

---

## 2. Inspection Workflow

1. Inspection created from Permit
2. Inspector assigned
3. Inspection scheduled
4. Inspection performed
5. Results recorded
6. Documents attached
7. Inspection marked Complete
8. Approval triggered (if required)

---

## 3. Approval → Invoice Workflow

1. Approval initiated by system or user
2. Approver reviews associated entity
3. Approval granted
4. Invoice generated
5. Invoice issued to Organization
6. Invoice status tracked (Open / Paid / Void)
